1. Project Overview

What is AAMSME?

AAMSME (Assessment Agency Management System for MSME) is a comprehensive web-based management system designed to streamline the assessment and certification process for MSME (Micro, Small and Medium Enterprises) training centers and assessment agencies.

Key Objectives

  • Centralize qualification and module management
  • Streamline exam scheduling and approval processes
  • Manage student registrations and progress tracking
  • Provide comprehensive reporting and analytics
  • Ensure data security and role-based access control

Target Users

  • Assessment Agency (AA): Primary administrators who manage qualifications, modules, and oversee the entire system
  • TC Admin: Training center administrators who manage their centers and schedules
  • TC Head: Training center heads who approve schedules and manage faculty
  • Exam Cell: Personnel responsible for exam coordination and scheduling
  • TC Faculty: Teachers who manage attendance, progress, and student interactions

5. Qualification Management

Assessment Agency Responsibilities

Creating Qualifications

  1. Access: Login as Assessment Agency → Qualification List
  2. Add Qualification: Click "Add Qualification" button
  3. Fill Details:
    • Qualification Name (e.g., "Automobile Service Technician")
    • NQR Number (National Qualification Register)
    • Sector (e.g., "Automotive")
    • Level (e.g., "Level 4")
    • Type (e.g., "Certificate")
    • Total Hours
  4. Save: Click "Add Qualification" to save

Managing Modules

  1. Access: Qualification Modules section
  2. Add Module: Click "Add Module" button
  3. Fill Details:
    • Module Name (e.g., "Engine Maintenance")
    • NOS Code (National Occupational Standard)
    • Optional Module (checkbox)
    • Hours (training duration)
    • Credit (credit points)
  4. Special Note: NOS Code "NA" can be duplicated for multiple modules

Mapping Modules to Qualifications

  1. Access: Qualification List → Click "Map Modules" on any qualification
  2. Select Modules: Check the modules that belong to this qualification
  3. Save Mapping: Click "Save Module Mappings"
  4. Verify: Use "View Modules" to see mapped modules

Important Notes:

  • Only Assessment Agency can create/edit qualifications and modules
  • NOS Code "NA" is allowed for duplicate entries
  • Modules must be mapped to qualifications before scheduling
  • CSV import is available for bulk data entry

6. Exam Scheduling Process

Complete Scheduling Workflow

Schedule Creation (TC Faculty)

  1. Login: Access system as TC Faculty
  2. Navigate: Go to Exam Schedules section
  3. Create Schedule: Click "Create New Schedule"
  4. Fill Basic Details:
    • Schedule Title
    • Qualification Name
    • Training Center
    • Exam Date and Time
    • Venue Details
  5. Select Modules: Choose modules to be examined
  6. Upload Files: Attach required documents
  7. Submit: Click "Create Schedule"

Exam Cell Review

  1. Notification: Exam Cell receives new schedule from TC Faculty
  2. Logistics Check: Verify exam resources availability
  3. Coordination: Ensure proper exam setup
  4. Review: Check technical requirements and staff availability
  5. Forward: Send to TC Head for internal approval

Internal Approval (TC Head)

  1. Notification: TC Head receives schedule from Exam Cell
  2. Review: Access schedule details and review
  3. Check: Verify all information is correct
  4. Approve/Reject: Click "Approve" or "Reject" with comments
  5. If Rejected: Schedule returns to TC Faculty for correction

Final Approval (Assessment Agency)

  1. Notification: Assessment Agency receives approved schedule from TC Head
  2. Review: Assessment Agency reviews complete schedule
  3. Validate: Check compliance with standards
  4. Approve/Reject: Final decision with comments
  5. If Approved: Generate file number and activate schedule
  6. Notification: All stakeholders notified of approval

Schedule Status Tracking

  • Draft: Created by TC Faculty, pending exam cell review
  • Pending Exam Cell: Submitted for logistics review
  • Pending TC Head: Exam cell approved, under internal approval
  • Pending AA: TC Head approved, under final Assessment Agency review
  • Approved: Fully approved with file number generated
  • Rejected: Rejected at any stage, requires resubmission

7. Approval Workflow

Detailed Approval Process

Stage Responsible Role Actions Timeline
Schedule Creation TC Faculty Create schedule with all details As needed
Exam Cell Review Exam Cell Check logistics and resources 24-48 hours
Internal Review TC Head Review and approve/reject 24-48 hours
Final Approval Assessment Agency Final validation, approval and file number generation 48-72 hours

Approval Criteria

TC Head Approval Criteria

  • Schedule information completeness
  • Module selection accuracy
  • Date and time feasibility
  • Venue availability
  • Faculty assignment verification

Exam Cell Approval Criteria

  • Exam resource availability
  • Logistics coordination
  • Technical requirements
  • Staff availability
  • Compliance with exam standards

Assessment Agency Approval Criteria

  • Qualification compliance
  • Module mapping accuracy
  • Standard adherence
  • Documentation completeness
  • Overall process compliance

Rejection Process

If a schedule is rejected at any stage:

  1. Rejection reason is recorded with comments
  2. Schedule returns to previous stage
  3. TC Faculty can modify and resubmit
  4. Process restarts from the rejection point
  5. All stakeholders are notified of changes
